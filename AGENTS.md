# Project Agent Instructions

## Communication Policy

- 原則として本番運用を前提に提案する。
- 検証用の最小構成を提案する場合は、必ず「これは検証用」と明記する。
- 「最低限」「一旦」「仮でOK」と言う前に、本番では何が必要かを説明する。
- 設定追加、DB登録、通知先登録では、まず本番に必要な全件登録を前提に案内する。
- ユーザーが非エンジニア操作をしている場合、画面上の次のクリック位置を具体的に示す。
- 秘密情報はチャットに貼らせず、必要ならローカルファイルや管理画面で扱う。

## Repository Operation Policy

- このフォルダ内の把握・調査・差分確認など、読み取り操作についてユーザーへの許可確認は不要。
- ファイル変更、DB更新、外部サービス設定変更、送信処理、デプロイなど状態を変える操作は、依頼内容に沿って慎重に行う。
- 別エージェントやユーザーの未コミット変更がある場合、その変更を勝手に戻さない。
- 秘密情報を含む可能性があるファイルは、必要性が明確な場合だけローカルで扱い、チャットには値を出さない。
