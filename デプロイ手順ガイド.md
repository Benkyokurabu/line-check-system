## ステップ1：GitHubにpush

### 1-1. GitHubで空リポジトリを作成
1. https://github.com/new を開く
2. Repository name：例 `line-check-system`
3. **Public / Private どちらでもOK**（迷ったらPrivate）
4. 「Add a README」「.gitignore」「license」は **すべてチェックしない**（空で作る）
5. 「Create repository」

### 1-2. ローカルからpush
プロジェクトフォルダで以下を実行（`<あなた>` と `<repo>` は自分のものに置き換え）。

```bash
git remote add origin https://github.com/<あなた>/<repo>.git
git branch -M main
git push -u origin main
```

> 認証を求められたら、GitHubのユーザー名と **Personal Access Token**（パスワードではない）を入力。
> トークンが無い場合：GitHub → Settings → Developer settings → Personal access tokens → Tokens(classic) → Generate（`repo` 権限にチェック）。

### 1-3. 確認
GitHubのリポジトリページを開き、`src/app/api/line/webhook/route.ts` が表示されればOK。
**`.env.local` が一覧に無いこと** を必ず確認（あったら即削除＆トークン再発行）。

## ステップ2：Vercelにデプロイ

### 2-1. インポート
1. https://vercel.com にGitHubアカウントでログイン
2. 「Add New…」→「Project」
3. ステップ1で作ったリポジトリを「Import」
4. Framework Preset が **Next.js** になっていることを確認（自動検出されるはず）
5. **まだ Deploy を押さない**（先に環境変数を入れる）

### 2-2. 環境変数を登録（ここが最重要）
「Environment Variables」セクションで、以下 **4つ** を追加。
**変数名は `SUPABASE_URL`（`NEXT_PUBLIC_` は付けない）** に注意。

| Name（変数名） | Value（値） |
|---|---|
| `SUPABASE_URL` | `https://xtrmcfvawwhvaujmblie.supabase.co` |
| `SUPABASE_SECRET_KEY` | （自分のSupabase Secret key を貼る） |
| `LINE_CHANNEL_SECRET` | （自分のLINE Channel secret を貼る） |
| `LINE_CHANNEL_ACCESS_TOKEN` | （自分のLINE Channel access token を貼る） |

- Environment は **Production**（最低限）にチェック。Preview/Developmentにも入れておくと後で楽。
- `.env.local` と同じ値を使う。値が分からなくなったら `.env.local` を開いて確認（チャットには貼らない）。

### 2-3. デプロイ
「Deploy」を押す。完了すると `https://<アプリ名>.vercel.app` のURLが発行される。
→ **このURLをメモ**（報告事項①で使う）。

### 2-4. 確認
`https://<アプリ名>.vercel.app` をブラウザで開いて、Next.jsの初期画面が表示されればOK。

> ※環境変数を後から変更/追加した場合は、Vercelで **再デプロイ（Redeploy）** が必要。

## メモ：技術的な前提（つまずいたとき用）

- Supabase接続用の環境変数名は **`SUPABASE_URL`** に統一済み（コード側も対応済み）。
- `line_messages` の本文カラムは **`text`**（`message_text` ではない）。`message_type` / `direction='inbound'` / `received_at` にマップ済み。reply_token等は `raw_event` に丸ごと保存。
- 重複防止は `line_message_id` のunique制約 + `upsert(ignoreDuplicates)` で実装済み。
- 自動返信・AI判定・タスク抽出は **第2段階**（今回はやらない）。
